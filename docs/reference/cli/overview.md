# brainpick overview

`brainpick overview [--root DIR]` prints one screen of the whole brain: the
bundle name, document/tag/entity counts, tier availability, and the index tree
with one-sentence descriptions. Add `--json` for the raw MCP payload. It is the
orientation call — the progressive-disclosure root — and, like the other read
mirrors, never compiles.

It mirrors [brain_overview](../mcp/brain-overview.md); the counts and tiers it
shows are [the tiers](../../the-tiers.md). Back to [CLI reference](../../reference-cli.md).
